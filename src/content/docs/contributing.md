Thanks for helping improve `vimcolorschemes`.

## Before you start

Choose the path that matches your change:

- Use the [frontend path](/getting-started/frontend/) for app UI and website behavior.
- Use the [full stack path](/getting-started/full-stack/) for Worker, database, and generated data changes.
- Read [Architecture](/architecture/) if you are unsure where a change belongs.

## Open a pull request

1. Fork the relevant repository.
2. Create a branch for your change.
3. Make the change.
4. Run the relevant local checks.
5. Open a pull request with a description and validation notes.

## What to include

In your pull request, include:

- The problem you are solving.
- The part of the system you changed.
- The checks you ran.
- Screenshots for visible UI changes.
- Notes about database, Worker, or generated data changes.

## Repository links

- App: [vimcolorschemes/vimcolorschemes](https://github.com/vimcolorschemes/vimcolorschemes)
- Worker: [vimcolorschemes/worker](https://github.com/vimcolorschemes/worker)
- Extractor: [vimcolorschemes/extractor.nvim](https://github.com/vimcolorschemes/extractor.nvim)
- Docs: [vimcolorschemes/docs](https://github.com/vimcolorschemes/docs)

If local setup fails, open an issue in the repository you are working on.
